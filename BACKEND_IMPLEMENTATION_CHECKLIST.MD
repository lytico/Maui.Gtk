# Platform.Maui.Linux.Gtk4 ‚Äî Backend Implementation Checklist

Implementation status of the .NET MAUI backend for Linux using GTK4. Adapted from the generic MAUI backend checklist.

> ‚úÖ = Implemented  |  ‚öÝÔ∏è = Partial  |  ‚ùå = Not implemented  |  N/A = Not applicable to Linux/GTK4

---

## 1. Core Infrastructure

### Platform Abstractions
- [x] **Platform View Type** ‚Äî `Gtk.Widget` as the native view abstraction via `GtkViewHandler<TVirtualView, TPlatformView>`
- [x] **Platform Context** ‚Äî `GtkMauiContext` with DI scope, handler factory, window scope management
- [x] **Dispatcher** ‚Äî `GtkDispatcherProvider` using `GLib.Functions.IdleAdd` for UI thread marshalling
- [x] **Event System** ‚Äî GTK4 signal/event model for user interactions (clicks, input, gestures)
- [x] **Handler Factory Integration** ‚Äî All handlers registered via `ConfigureMauiHandlers` in `AppHostBuilderExtensions`
- [x] **App Host Builder Extension** ‚Äî `UseLinuxGtkHandlers()` wires up all handlers and services

### Rendering Pipeline
- [x] **View Renderer** ‚Äî `GtkViewHandler` base traverses virtual view tree, creates/connects GTK widgets
- [x] **Property Change Propagation** ‚Äî `IPropertyMapper` re-maps views when `IView` property changes fire
- [x] **Child Synchronization** ‚Äî `GtkLayoutPanel` handles add/remove/reorder of child views via `CustomLayout` (P/Invoke `gtk_custom_layout_new`, replacing `Gtk.Fixed`'s `FixedLayout`)
- [x] **Style/Attribute Application** ‚Äî CSS-based styling via `ApplyCss()` helper for colors, fonts, sizes
- [x] **Layout Negotiation** ‚Äî `GtkLayoutPanel` uses `SetArrangedSize` + `ApplyArrangedSizes` (IdleAdd priority 121) to override GTK's minimum-based allocation with MAUI-arranged sizes

### Native Interop
- [x] **GTK4 Signal Handling** ‚Äî Native GTK signal connections for widget events
- [x] **Gesture Controllers** ‚Äî `Gtk.GestureClick`, `Gtk.GestureDrag`, `Gtk.EventControllerMotion`
- [x] **Keyboard Event Handling** ‚Äî GTK `Gtk.EventControllerKey` for Entry/Editor key events
- [x] **Scroll Handling** ‚Äî `Gtk.ScrolledWindow` for scroll containers, P/Invoke for scroll policy (`gtk_scrollable_set_vscroll_policy`)
- [x] **P/Invoke Interop** ‚Äî Direct native calls for fontconfig (`libfontconfig.so.1`), Pango (`libpangocairo-1.0.so.0`, `libpangoft2-1.0.so.0`), Cairo (`libcairo.so.2`), and GTK4 custom layout/scrollable APIs

### Build & Packaging
- [x] **MSBuild Targets** ‚Äî `buildTransitive/` props and targets for MauiAsset, MauiImage, MauiIcon, MauiFont, MauiSplashScreen resource deployment
- [x] **Hicolor Icon Theme** ‚Äî Linux icon theme integration for app icons via build targets
- [x] **Project Templates** ‚Äî `dotnet new` template (`maui-linux-gtk4-app`) with App.cs, MainPage.cs, MauiProgram.cs, Program.cs scaffolding
- [x] **XAML Support** ‚Äî Runtime XAML page loading works (demonstrated by XamlRuntimePage.xaml with code-behind)

## 12. Common View Properties (Base Handler)

Every handler must support these properties mapped from the base `IView`:

### Visibility & State
- [x] Opacity ‚Äî `MapOpacity` applies via CSS
- [x] IsVisible ‚Äî `MapVisibility` toggles `Gtk.Widget.SetVisible()`
- [x] IsEnabled ‚Äî `MapIsEnabled` toggles `Gtk.Widget.SetSensitive()`
- [x] InputTransparent ‚Äî `MapInputTransparent` sets `Gtk.Widget.SetCanTarget()`

### Sizing
- [x] WidthRequest / HeightRequest ‚Äî Used in `GetDesiredSize()` measurement
- [x] MinimumWidthRequest / MinimumHeightRequest ‚Äî Used in measurement
- [x] MaximumWidthRequest / MaximumHeightRequest ‚Äî Constrained in `GetDesiredSize()`

### Layout
- [x] HorizontalOptions / VerticalOptions ‚Äî Handled by MAUI cross-platform layout
- [x] Margin ‚Äî Handled by MAUI layout engine (cross-platform)
- [x] Padding ‚Äî Mapped on individual handlers (Entry, Editor, Button, etc.)
- [x] FlowDirection (LTR, RTL) ‚Äî `MapFlowDirection` sets `Gtk.TextDirection`
- [x] ZIndex ‚Äî `MapZIndex` reorders GTK siblings via `InsertBefore`/`InsertAfter`

### Appearance
- [x] BackgroundColor / Background ‚Äî `MapBackground` applies via CSS `background-color`

### Transforms
- [x] TranslationX / TranslationY ‚Äî `Gsk.Transform.Translate()` via `SetChildTransform`
- [x] Rotation / RotationX / RotationY ‚Äî `Gsk.Transform.Rotate()` via `SetChildTransform`
- [x] Scale / ScaleX / ScaleY ‚Äî `Gsk.Transform.Scale()` via `SetChildTransform`
- [x] AnchorX / AnchorY ‚Äî Translate-to-anchor pattern in transform chain

### Effects
- [x] Shadow ‚Äî `MapShadow` applies via CSS `box-shadow`
- [x] Clip ‚Äî `MapClip` applies CSS `border-radius` + `overflow:hidden` for RoundRectangleGeometry/EllipseGeometry

### Automation
- [x] AutomationId ‚Äî `MapAutomationId` sets accessible name
- [x] Semantic properties ‚Äî `MapSemantics` sets accessibility description

### Interactivity Attachments
- [x] **ToolTip** ‚Äî `ToolTipProperties.Text` mapped to `Gtk.Widget.SetTooltipText()`
- [x] **ContextFlyout** ‚Äî `FlyoutBase.GetContextFlyout()` ‚Üí popup menu on right-click via `Gtk.PopoverMenu` + `GestureClick(button=3)`

> Note: All handler Mapper fields must chain from `GtkViewHandler.ViewMapper` (not `ViewHandler.ViewMapper`) to inherit transform, opacity, shadow, clip, tooltip, and context flyout property mappings.
